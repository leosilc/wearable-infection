# NightSignal Robustness Study — Missing Data Analysis

This repository extends the [NightSignal algorithm](https://github.com/StanfordBioinformatics/wearable-infection) to evaluate its robustness under simulated missing sensor data from Fitbit wearables. The study is part of an undergraduate research project investigating pre-symptomatic and assymptomatic COVID-19 detection.

---

## Project Overview

The pipeline has three stages:

```
Raw Fitbit data
      ↓  run_all.py
Processed CSVs + original NightSignal results
      ↓  generate_missing.py
CSVs with simulated missing data + NightSignal results per scenario
      ↓  results.py
Metrics (confusion matrix, recall, precision, accuracy) + plots
```

## Data Structure

Place raw patient data in `data/future/` (kept locally, not committed to the repository). Move only the patients you want to process into `data/raw/`:

```
data/
├── future/                        ← all raw patient data (not committed)
│   └── {patient_id}/
│       ├── Orig_Fitbit_HR.csv
│       └── Orig_Fitbit_ST.csv
├── raw/                           ← patients selected for current run
│   └── {patient_id}/
│       ├── Orig_Fitbit_HR.csv
│       └── Orig_Fitbit_ST.csv
├── processing/                    ← generated by run_all.py (do not edit)
│   └── {patient_id}/
│       └── {patient_id}_temp.csv
└── missing/                       ← generated by generate_missing.py
    └── MCAR/
        └── {rate}pct/
            └── {patient_id}/
                └── {patient_id}_temp.csv
```

## Step-by-Step Pipeline

### Step 1 — Select patients

Copy the patient folders you want to test from `data/future/` into `data/raw/

### Step 2 — Process raw data and run NightSignal (original)

Merges HR and Steps CSVs into a single minute-level file, then runs NightSignal on each patient.

```bash
python3 run_all.py
```

**Output:**
```
results/
└── {patient_id}/
    ├── {patient_id}_signals.json   ← alert states (0=green, 1=yellow, 2=red)
    └── {patient_id}_plot.pdf       ← RHR timeline with alerts
```

---

### Step 3 — Configure missing data scenarios

Open `generate_missing.py` and set the rates you want to test:

```python
# Line 14
RATES = [10, 30, 50]   # missing rates in % — edit as needed
SEED  = 42             # keep fixed for reproducibility
```

---

### Step 4 — Generate missing data and run NightSignal on each scenario

Applies MCAR (Missing Completely At Random) to the HR column, saves the degraded CSVs, and automatically runs NightSignal on each one.

```bash
python3 generate_missing.py
```

**Output:**
```
data/missing/
└── MCAR/
    └── {rate}pct/
        └── {patient_id}/
            └── {patient_id}_temp.csv   ← CSV with missing HR values

results/
└── {patient_id}/
    └── missing/
        └── MCAR/
            └── {rate}pct/
                ├── {patient_id}_signals.json
                └── {patient_id}_plot.pdf
```

> **Note:** The degraded CSV keeps the original HR and Steps values in two extra columns (`HR_MISSING`, `ST_MISSING`) for comparison. NightSignal only reads the first four columns and ignores them.

---

### Step 5 — Calculate metrics and generate plots

Compares each missing-data scenario against the original result for every patient. Treats the original NightSignal output as ground truth.

```bash
python3 results.py
```

**Output:**
```
results/
├── metrics.csv                         ← all metrics in a single table
├── accuracy_plot.png                   ← boxplot of accuracy by rate
└── {patient_id}/
    └── {patient_id}_metrics.txt        ← confusion matrix + metrics per scenario
```
